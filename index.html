7장 — 집계 쿼리
GROUP BY · 집계함수 · WITH ROLLUP · HAVING

GROUP BY

SELECT에 올 수 있는 것: GROUP BY에 쓴 칼럼 + 집계함수만
GROUP BY에 없는 일반 칼럼 → 오류 or 엉뚱한 값
문법 포인트예시
칼럼 순번 사용 가능GROUP BY 1, 2 (= 첫째, 둘째 칼럼)
표현식도 가능GROUP BY SUBSTRING(district,1,2)
DISTINCT와 차이DISTINCT = 중복 제거만 / GROUP BY = 집계함수와 함께

집계 함수 ★

함수의미NULL 처리
COUNT(*)전체 행 수NULL 포함
COUNT(칼럼)값 있는 행 수NULL 제외!
COUNT(DISTINCT 칼럼)중복 제거 후 개수NULL 제외
SUM / AVG / MAX / MIN합·평균·최대·최소AVG도 NULL 제외
VAR_POP / STDDEV_POP분산·표준편차—

WITH ROLLUP ★ 시험

동작

GROUP BY 뒤에 붙이면
소계 + 총계 행 자동 추가
총계 행의 칼럼 값 = NULL

GROUPING() 함수

총계 행 구분할 때 사용
반환 1 = 총계 행
반환 0 = 일반 집계 행 
GROUP BY movie_type WITH ROLLUP
-- NULL 대신 '전체합계' 표시
SELECT IF(GROUPING(movie_type)=1, '전체합계', movie_type), SUM(sale_amt)
FROM box_office WHERE YEAR(release_date)=2019
GROUP BY movie_type WITH ROLLUP;

HAVING vs WHERE ★★ 자주 틀림

WHEREHAVING
시점집계 전집계 후
위치GROUP BY 앞GROUP BY 뒤
쓸 수 있는 조건일반 칼럼집계함수, GROUPING()
WHERE에 COUNT() 같은 집계함수 쓰면 오류! → HAVING으로 이동
8장 — 조인 (Join)
INNER · LEFT/RIGHT OUTER · NATURAL · CROSS · UNION

INNER JOIN ★★

양쪽 테이블에 모두 존재하는 데이터만. 한쪽에 없으면 결과에서 제외.
방법예시
INNER JOIN + ON (권장)FROM city a INNER JOIN country b ON a.countrycode = b.code
구식 쉼표 조인FROM city a, country b WHERE a.countrycode = b.code
INNER 생략 가능JOIN만 써도 INNER JOIN으로 동작

OUTER JOIN ★★

종류기준 테이블반대쪽 없으면
LEFT JOIN왼쪽(첫 번째)NULL로 채움
RIGHT JOIN오른쪽(두 번째)NULL로 채움
COUNT(*) vs COUNT(b.name) 차이:
COUNT(*)는 NULL 포함 전체 행 수 / COUNT(b.name)은 JOIN된 것(도시 있는 것)만

CROSS JOIN / NATURAL JOIN

종류설명주의
CROSS JOIN모든 조합 = m × n 행조인 조건 빠뜨리면 의도치 않게 발생!
NATURAL JOIN같은 칼럼명 자동 연결칼럼명·타입 일치 필수. 실무에서 잘 안 씀.

UNION ★★

필수 조건: 칼럼 수 동일 + 타입 동일 + ORDER BY는 맨 마지막에만
구문설명
UNION중복 제거 후 합치기
UNION ALL중복 포함 전체 (더 빠름)
조인 = 옆으로(칼럼 증가) / UNION = 아래로(행 증가)
9장 — 서브쿼리
스칼라 · 파생 테이블 · LATERAL · IN / EXISTS / ANY / ALL

위치별 분류 ★★★

위치이름반환핵심 제약
SELECT 절스칼라 서브쿼리단일 행·단일 값여러 행 반환 시 오류!
FROM 절파생 테이블여러 행·열 가능별칭(AS) 필수!
WHERE 절조건 서브쿼리연산자에 따라IN / EXISTS / ANY / ALL

스칼라 서브쿼리

-- 각 도시마다 그 나라 이름을 SELECT에서 가져오기
SELECT a.name, a.population,
       (SELECT b.name FROM country b WHERE a.countrycode = b.code) countryname
FROM city a;
-- ❌ 오류: 서브쿼리에서 여러 칼럼 반환
-- ❌ 오류: WHERE 조건 없어 여러 행 반환

파생 테이블 (FROM 절)

별칭 없으면 오류! 반드시 FROM (...) 별칭 형태
-- 먼저 계산한 결과를 테이블처럼 사용
SELECT a.dept_no, a.dept_name, mng.emp_no
FROM departments a,
     (SELECT b.dept_no, b.emp_no, c.first_name
        FROM dept_manager b, employees c
       WHERE b.emp_no = c.emp_no
         AND SYSDATE() BETWEEN b.from_date AND b.to_date
     ) mng  ← 별칭 필수
WHERE a.dept_no = mng.dept_no;

LATERAL ★★ 시험 출제

일반 파생 테이블LATERAL 파생 테이블
실행 방식메인쿼리와 독립 실행메인쿼리 행마다 함께 실행
바깥 테이블 참조❌ 불가 → 오류✅ 가능
-- ❌ 9-15: 서브쿼리 안에서 a.dept_no 참조 불가 → 오류
FROM departments a, (... AND a.dept_no = b.dept_no) mng

-- ✅ 9-16: LATERAL 붙이면 a.dept_no 참조 가능
FROM departments a, LATERAL (... AND a.dept_no = b.dept_no) mng

WHERE절 서브쿼리 — IN / EXISTS / ANY / ALL ★★★

연산자의미실질 동작
IN (...)이 중에 있으면—
NOT IN (...)이 중에 없으면서브쿼리에 NULL 있으면 결과 없음!
EXISTS (...)행이 하나라도 존재하면보통 IN보다 빠름
> ANY (...)하나라도 크면 OK= > MIN(...) 쉬운 조건
> ALL (...)전부보다 커야 OK= > MAX(...) 까다로운 조건
< ANY (...)하나보다 작으면 OK= < MAX(...)
< ALL (...)전부보다 작아야= < MIN(...)
암기: ANY = 가장 작은 값 기준 (MIN) / ALL = 가장 큰 값 기준 (MAX)
10장 — DML & 트랜잭션
INSERT · UPDATE · DELETE · TRUNCATE · COMMIT · ROLLBACK · SAVEPOINT

INSERT

방식구문
단일 행INSERT INTO t (c1,c2) VALUES (v1,v2)
다중 행VALUES (v1,v2), (v3,v4), ...
SELECT 결과 삽입INSERT INTO t SELECT ... FROM ... WHERE ...
있으면 수정·없으면 삽입INSERT ... ON DUPLICATE KEY UPDATE 칼럼=값
NOT NULL 칼럼 빠뜨리면 오류 / PK 중복 넣으면 오류

UPDATE

WHERE 없으면 전체 행이 수정됨!
방식구문 패턴
단일 테이블UPDATE t SET c1=v1, c2=v2 WHERE 조건
다중 테이블UPDATE t1 a, t2 b SET a.c1=b.c2 WHERE a.id=b.id
ORDER BY 가능UPDATE t SET ... WHERE ... ORDER BY c DESC

DELETE vs TRUNCATE vs DROP ★★★

DELETETRUNCATEDROP
분류DMLDDLDDL
범위WHERE로 일부 가능전체만테이블 자체 삭제
ROLLBACK✅ 가능❌ 불가❌ 불가
속도느림빠름—
DDL은 자동 COMMIT → ROLLBACK 불가!
-- ORDER BY + LIMIT 조합 가능
DELETE FROM emp WHERE emp_name='홍길동' ORDER BY emp_no DESC LIMIT 1;

-- 다중 테이블 삭제
DELETE a, b FROM t1 a, t2 b WHERE a.id = b.id;

트랜잭션 ★★★

명령어의미
COMMIT영구 반영
ROLLBACK전부 취소 (마지막 COMMIT 시점으로)
SAVEPOINT 이름중간 저장점 설정
ROLLBACK TO SAVEPOINT 이름그 지점 이후만 취소
START TRANSACTION자동커밋 있어도 수동 트랜잭션 시작
AUTOCOMMIT = 1 (기본)이면 DML 즉시 자동 커밋 → ROLLBACK 불가
START TRANSACTION 또는 SET autocommit=0으로 해제해야 ROLLBACK 가능 
START TRANSACTION;
SAVEPOINT A;
DELETE FROM emp_tran1 WHERE salary IS NULL;  -- 작업1
SAVEPOINT B;
DELETE FROM emp_tran1 WHERE emp_name='홍길동' LIMIT 1;  -- 작업2
ROLLBACK TO SAVEPOINT B;  -- 작업2만 취소, 작업1은 유지
COMMIT;                   -- 작업1 영구 반영
